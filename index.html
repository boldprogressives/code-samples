Radio Buttons and Checkboxes

Take a look at a standard radio/checkbox code. You can break it down into three sections.

<input type="radio" name="action_syria_usaction" value="yes" id="usaction_yes" /> <label for="usaction_yes">Yes</label> <input type="radio" name="action_syria_usaction" value="yes" id="usaction_yes" /> <label for="usaction_yes">Yes</label>

Input type determines whether you are working with "radio"/radio buttons or "checkbox"/checkboxes.

<input type="radio" name="action_syria_usaction" value="yes" id="usaction_yes" /> <label for="usaction_yes">Yes</label>

The "name" and the "value" are the parts of our code that get recorded into our database. Think of "name" as the question you're asking the member and "value" as the member's answer to the question.

To learn more about choosing a name, be sure to read the ActionKit/ActionFields Guidelines. (for example, names should ALWAYS start with the prefix action_; that's what tells the database it's time to start storing information.)

For questions where you want the member to only choose one answer ("Do you want ice cream? Yes, No, Maybe"), you should radio inputs with the same name. This ensures that a user can only select one option at a time. (But the different inputs still need distinct values, or else we won't know which one they picked!)

On the other hand, checkbox type inputs don't interact with one another, so a user can check as many as they want. For questions where you want the member to be able to choose multiple answers ("What topping do you want on the pizza? pineapple, pepperoni, tofu, mushrooms?"), you can use any distinct combinations of "name" and "value", but our usual approach is to use different "name"s and the value "true". There's some example code below.

<input type="radio" name="action_syria_usaction" value="yes" id="usaction_yes" /> <label for="usaction_yes">Yes</label>

The "id" and "label for" section makes it easier for our members to click on various options. For best practice, just squish together your "name" and "value" and make sure they're the same for "id' and "label for". So if <name="great" value="job">, then "id" and "label for" would be "great_job".

Radio Buttons

Q: Do you want ice cream?

<input type="radio" name="action_icecream" value="yes" id="icecream_yes" /> <label for="icecream_yes">Yes</label> <input type="radio" name="action_icecream" value="no" id="icecream_no" /> <label for="icecream_no">No</label> <input type="radio" name="action_icecream" value="na" id="icecream_na" /> <label for="icecream_na">I don't know.</label>

Checkboxes

Q: What kind of pizza toppings do you want to have on the pizza?

<input type="checkbox" name="action_pizza_topping" value="pineapple" id="pizza_topping_pineapple" /> <label for="pizza_topping_pineapple">Pineapple</label> <input type="checkbox" name="action_pizza_topping" value="pepperoni" id="pizza_topping_pepperoni" /> <label for="pizza_topping_pepperoni">Pepperoni</label> <input type="checkbox" name="action_pizza_topping" value="tofu" id="pizza_topping_tofu" /> <label for="pizza_topping_tofu">Tofu</label> <input type="checkbox" name="action_pizza_topping" value="mushrooms" id="pizza_topping_mushrooms" /> <label for="pizza_topping_mushrooms">Mushrooms</label>

Text

For this option, you can just copy and paste the below code and make the appropriate "name".

<textarea name="action_choose_a_name" rows="1" cols="15">Type your opinion</textarea>

Hidden option

To create a hidden option, include the onClick in the <label> section of your code. Make sure that it matches to a <div></div> where you will put the dropdown content. See below for code.

<input type="radio" name="action_icecream" value="na" id="icecream_na" onClick="$('.hiddenoption_name_goes_here-div').toggle(500);"/> <label for="icecream_na" >I don't know.</label> <div class="hiddenoption_name_goes_here-div" style="display:none; /"> Put your hidden option content here. </div>
